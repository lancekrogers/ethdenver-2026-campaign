# Architecture Documentation

System design, component diagrams, and technical specifications.

## What Goes Here

- System architecture diagrams
- Component specifications
- Data flow documentation
- Infrastructure design
- Security architecture
- Performance requirements

## Structure

```
architecture/
├── overview.md        # High-level system design
├── components/        # Individual component specs
├── diagrams/          # Mermaid, PlantUML, or images
└── decisions/         # Link to ADRs
```

## Guidelines

- Use diagrams to communicate complex systems
- Keep documentation updated with implementation
- Reference ADRs for decision rationale
- Include deployment and scaling considerations
