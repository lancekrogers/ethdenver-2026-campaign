# Architecture Decision Records

Documents significant architectural decisions with context and consequences.

## What Goes Here

- Architectural decisions (ADR format)
- Technology choices
- Design trade-offs
- Migration decisions

## ADR Format

Each ADR should follow this structure:

```markdown
# ADR-NNN: Title

## Status
[Proposed | Accepted | Deprecated | Superseded]

## Context
What is the issue we're facing?

## Decision
What is the change we're proposing?

## Consequences
What are the trade-offs?
```

## Naming Convention

- `0001-initial-architecture.md`
- `0002-database-selection.md`
- Sequential numbering, never reuse numbers
