# Task: Submit Hedera Track 3 Bounty

**Task Number:** 01 | **Sequence:** 02_submit_all_tracks | **Autonomy:** low

## Objective

Submit the Hedera Track 3 ($5k) bounty on the ETHDenver submission portal. Fill out the submission form with all required fields: project URL, demo video, architecture description, and README link.

## Requirements

- [ ] Submission form accessed on the ETHDenver portal
- [ ] All required fields filled completely
- [ ] All links included are verified accessible
- [ ] Submission confirmed (received confirmation from portal)
- [ ] Submission URL or confirmation ID recorded

## Implementation

### Step 1: Gather submission materials

Before opening the form, gather all links and descriptions:

| Material | Value |
|----------|-------|
| Project Repository URL | https://github.com/[org]/agent-coordinator |
| Demo Video URL | [YouTube URL from 08_demo_video] |
| README URL | [direct link to README.md] |
| Architecture Doc URL | [direct link to docs/architecture.md] |
| Dashboard URL | [deployed dashboard URL] |
| Demo Notes URL | [direct link to docs/demo-notes.md] |

### Step 2: Prepare description text

Write a submission description (check form character limits):

**Project Name**: [agent-coordinator project name]

**Short Description** (1-2 sentences): A multi-agent coordinator that uses Hedera Consensus Service (HCS) for agent-to-agent messaging and Hedera Token Service (HTS) for automated payments in a decentralized AI economy.

**Detailed Description** (1-2 paragraphs): Describe the system, emphasizing native Hedera service usage. Reference the architecture document for technical depth. Mention the live dashboard and demo video.

**Bounty Alignment**: Explicitly state how this meets Hedera Track 3 requirements for native Hedera service usage. Reference HCS topic creation, message publishing, message subscription, HTS token transfers, and the Go SDK.

### Step 3: Fill out the submission form

Navigate to the ETHDenver submission portal. Fill out each field:

1. **Project Name**: Enter the project name
2. **Repository URL**: Paste the GitHub repo URL
3. **Demo Video**: Paste the YouTube URL
4. **Description**: Paste the prepared description
5. **Team Members**: Add team member information
6. **Bounty Track**: Select Hedera Track 3
7. **Additional Links**: Add dashboard URL, architecture doc, etc.

### Step 4: Review before submitting

Before clicking submit:

- [ ] Re-read every field for typos
- [ ] Verify every link opens correctly in a new tab
- [ ] Confirm the correct bounty track is selected
- [ ] Check that the description fits within character limits

### Step 5: Submit and record confirmation

- Click submit
- Record the confirmation (screenshot, confirmation ID, or confirmation email)
- Note the submission URL if the portal provides one

## Done When

- [ ] Submission form completed with all fields
- [ ] All links verified accessible
- [ ] Submission confirmed on portal
