# 10 — Growth Mechanics & Incentive Design

## Problem Statement

The original design assumes creators and users will show up organically. They won't. Every successful crypto platform (Curve, Lido, Compound, Jupiter) spent significant capital and design effort on incentive loops to bootstrap supply (creators) and demand (depositors) simultaneously.

This document addresses every growth gap identified in the critical review.

## 1. Referral System

### Mechanism

Every wallet gets a referral code. When a referred user deposits, the referrer earns a percentage of platform fees generated by that user's capital — permanently.

```
Referral Fee Structure:
  Tier 1: Direct referral → 10% of platform fees from referred user's capital
  Tier 2: Referred user refers someone → 3% of fees from that user's capital
  Max depth: 2 levels

Example:
  Alice refers Bob. Bob deposits $10K into an agent.
  Agent trades $500/day. Platform fee at 1%: $5/day.
  Alice earns: $0.50/day ($182.50/year) — forever, as long as Bob's capital is active.

  Bob refers Carol. Carol deposits $5K.
  Agent trades $250/day. Platform fee: $2.50/day.
  Alice earns: $0.075/day (3% of Carol's fees)
  Bob earns: $0.25/day (10% of Carol's fees)
```

### On-Chain Implementation

```rust
#[account]
pub struct ReferralState {
    pub referrer: Pubkey,           // who referred this user
    pub referrer_l2: Option<Pubkey>, // who referred the referrer
    pub referred_users: u32,         // count of direct referrals
    pub total_earned: u64,           // lifetime referral earnings
    pub bump: u8,
}

// On every platform fee collection:
fn distribute_referral_fees(fee: u64, depositor: &Pubkey) -> Result<()> {
    let referral = get_referral_state(depositor)?;
    if let Some(referrer) = referral.referrer {
        let l1_share = fee * 1000 / 10_000; // 10%
        transfer_to(referrer, l1_share)?;
    }
    if let Some(referrer_l2) = referral.referrer_l2 {
        let l2_share = fee * 300 / 10_000; // 3%
        transfer_to(referrer_l2, l2_share)?;
    }
    Ok(())
}
```

### Bonus Share Mechanics

```rust
fn calculate_deposit_shares(amount: u64, vault: &VaultState, incentive: &IncentiveConfig) -> u64 {
    let base_shares = amount * vault.total_shares / vault.total_nav;

    let bonus_multiplier = if Clock::get()?.unix_timestamp < incentive.bonus_deadline {
        11_500 // 1.15x = 15% bonus
    } else {
        10_000 // 1.0x = no bonus
    };

    base_shares * bonus_multiplier / 10_000
}

// Creator still gets their ownership % of the TOTAL shares (including bonus)
// Platform absorbs the dilution cost, not the creator
```

## 3. Creator Incentive Tiers

### The Problem

A developer with a profitable Polymarket bot has no reason to join our platform — they keep 100% of profits running it themselves. We need to offer something they can't get alone.

### What Creators Can't Do Alone

1. **Raise external capital** — they trade with their own money. We let them manage other people's money with smart contract custody (legal-ish fund management).
2. **Marketplace discovery** — nobody finds their bot. We surface them to depositors.
3. **Infrastructure** — multi-platform adapters, risk management, NAV calculation, cross-chain bridging, compliance-adjacent features.
4. **Credibility** — "I run a bot" vs "My agent manages $500K on OBEY Platform with verified on-chain returns."

### Creator Revenue Model (Revised)

Replace the simple "ownership %" with a multi-layered incentive:

| Tier | AUM Threshold | Creator Gets | Platform Gets |
|------|-------------|-------------|---------------|
| **Seed** | $0 - $50K | 15% ownership shares + 0% fee share | 100% of platform fees |
| **Growth** | $50K - $250K | 15% ownership + 20% of platform fees | 80% of platform fees |
| **Scale** | $250K - $1M | 15% ownership + 35% of platform fees | 65% of platform fees |
| **Elite** | $1M+ | 15% ownership + 50% of platform fees | 50% of platform fees |

**Example at $500K AUM (Scale tier):**
```
Daily volume: $25K (5% turnover)
Platform fee (1%): $250/day
Creator fee share (35%): $87.50/day
Platform keeps: $162.50/day

Creator annual income: $31,937 (fee share) + ownership share appreciation
  vs. running their own bot: $0 from external capital (only their own money)

If creator has $50K of their own capital making 20% APY = $10K/year
  Platform gives them: $31,937 + share of $500K pool growth
  Net: 3-5x better than solo
```

## 4. Fee Structure (Revised)

### Original: 1-2% flat fee per trade

**Problem:** Too expensive vs. direct Polymarket trading (0.1%), not competitive enough to justify platform value.

### Revised: Tiered Performance-Aligned Fee

```
Platform Fee = Base Fee + Management Fee + Performance Fee

Base Fee:         0.5% of trade value (covers infrastructure)
Management Fee:   0.3% of trade value (covers agent operation)
Performance Fee:  10% of profits above high-water mark (quarterly)

Total cost to user:
  If agent is profitable: ~0.8% per trade + 10% of profits
  If agent is flat/losing: ~0.8% per trade + $0 performance fee

Compared to:
  Polymarket direct: 0.1% per trade
  Enzyme Finance: 0.5-2% management + 0-20% performance
  Traditional hedge fund: 2% management + 20% performance
  OBEY: 0.8% effective + 10% performance — competitive with DeFi vaults
```

### Fee Implementation

```rust
fn calculate_fees(trade_value: u64, vault: &VaultState) -> (u64, u64) {
    // Base + management: 80 bps (0.8%)
    let trade_fee = trade_value * 80 / 10_000;

    // Performance fee: calculated quarterly on NAV update
    // Not per-trade — assessed on NAV growth above high water mark
    // Handled in separate quarterly_settlement instruction

    (trade_fee, 0) // performance fee calculated separately
}

fn quarterly_performance_settlement(vault: &mut VaultState) -> u64 {
    if vault.total_nav <= vault.high_water_mark {
        return 0; // no performance fee if NAV hasn't exceeded previous high
    }

    let profit = vault.total_nav - vault.high_water_mark;
    let performance_fee = profit * 1000 / 10_000; // 10%

    // Mint performance fee as new shares to platform treasury
    // This dilutes existing shares proportionally
    vault.high_water_mark = vault.total_nav;

    performance_fee
}
```
