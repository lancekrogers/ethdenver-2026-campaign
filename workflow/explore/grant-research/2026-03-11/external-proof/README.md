This is the standard for application-facing and public-facing docs.

- social copy
- step-by-step submission guides

## Claim Standard

A claim is safe for this track when it is backed by one or more of:

- on-chain transaction links
- deployed contract addresses
- repository code paths
- passing tests
- official ecosystem program docs

## Allowed Phrasing

- "implemented in code"
- "verified on-chain"
- "documented in tests"
- "remaining proof gap"
- "designed for"

## Avoid Unless Directly Proven

- "production-grade"
- "fully proven end-to-end"
- "self-sustaining" as a demonstrated fact
- "already live" when only deployments exist
